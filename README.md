# Michael Hale's Website

A website for Michael Hale that showcases featured projects, technical skills, and more.

## Stack

- Next.js App Router
- React 19
- Tailwind CSS
- MDX content from `content/work`
- YAML technology metadata from `content/technologies`
- Dockerized production runtime

## Setup

```bash
git clone git@github.com:mjhale/michaelhale.com.git
cd michaelhale.com
corepack enable
pnpm install
pnpm dev
```

App runs at [http://localhost:3000](http://localhost:3000).

## Scripts

- `pnpm dev`: Sync assets, start local Next dev server with Turbopack
- `pnpm build`: Sync assets, build production app with Turbopack
- `pnpm start`: Sync assets, run production server
- `pnpm lint`: Run ESLint

## Content model

- Work entries: `content/work/**/**/*.mdx`
- Technology records: `content/technologies/*.yml`
- Public assets are generated by `scripts/sync-assets.mjs` into `/public` before dev/build/start

## Deployment

The application is containerized using Podman and supports multi-platform builds.
For normal builds, avoid `--no-cache` so Podman can reuse build layers. Pre-generating
assets before a multi-platform build keeps the expensive image sync work warm:

```bash
# Pre-generate public assets
pnpm prebuild

# Build for multiple platforms
podman build --platform linux/amd64,linux/arm64 \
  --manifest ghcr.io/mjhale/michaelhale.com:latest .

# Push latest
podman manifest push --all \
  ghcr.io/mjhale/michaelhale.com:latest \
  docker://ghcr.io/mjhale/michaelhale.com:latest

# Push git commit tag
podman manifest push --all \
  ghcr.io/mjhale/michaelhale.com:latest \
  docker://ghcr.io/mjhale/michaelhale.com:$(git rev-parse HEAD)

# Run container
podman run -d -p 3000:3000 --env-file .env \
  --name michaelhale.com ghcr.io/mjhale/michaelhale.com:latest
```

Asset sync uses `SYNC_ASSETS_CONCURRENCY=4` by default. Tune that value for your
machine if image generation competes too heavily with other work.
